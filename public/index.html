Why Docker?

Your AI agent has filesystem access, API keys, and full brain memory. Docker isn't optional -- it's the foundation of trust.

  • ✓ Agent isolated from host -- The AI process runs in its own container. It can only see what you explicitly mount. No surprises.
  • ✓ Memory sandboxed -- Each tenant's brain (facts, wiki, soul) lives in an isolated volume. Cross-contamination is architecturally impossible.
  • ✓ API keys contained -- Secrets are injected via environment variables and never written to disk. Rotate keys without touching code.
  • ✓ Easy rollback -- Every deployment is a tagged image. Something breaks? Roll back in seconds with a single command.
$ docker run -d \
  --name businesslog \
  -p 3000:3000 \
  -v ./brain:/app/brain \
  -v ./secrets:/run/secrets \
  --env-file .env \
  businesslog-ai:latest

✓ Container started
✓ Brain loaded (847 facts, 23 wiki pages)
✓ WebSocket listening on ws://0.0.0.0:3000
✓ Agent ready.
